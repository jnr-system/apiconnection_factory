---
description: 新しいAPI連携スクリプトフォルダの初期セットアップ
---

新しいスクリプトフォルダ（`scripts/` 配下）を作成・追加した際は、以下の手順を必ず実行してください。

1. **環境変数のリファクタリング**
   - 対象の `.py` ファイルを読み込み、ベタ書きされている APIトークン（`RAKURAKU_TOKEN`など）や、JSONファイルパス（`google_secret.json`など）を `os.environ` や `.env` からの読み込みに修正する。
   - `python-dotenv` の `load_dotenv()` ブロックをファイルの先頭に追加する。
   - **【重要】** 実装の偏りを防ぐため、`RR_to_SS_zaiko` フォルダだけでなく、他のフォルダ（例：`RR_to_SS_seiyakuprocess` や `tenki_to_SS_tempreture` など）から **2〜3個のファイル** を参照し、プロジェクト全体の標準的な環境変数の扱い方に合わせてリファクタリングを行うこと。

2. **`.env` ファイルの作成**
   - ローカル実行用の `.env` ファイル（ベタ書きされていた実際の認証情報を含む）を該当フォルダに作成する。
   - ※ `.env_template` ファイルの作成は不要です。

3. **`README.md` の作成**
   - 該当フォルダ内に `README.md` を作成する。
   - スクリプトの概要、対象としているDB、環境変数（必須項目）、およびログの出力場所などの実行方法を他のフォルダのフォーマットに合わせて記載する。

4. **ルートの `README.md` の更新**
   - プロジェクト直下（`apiconnection_factory/README.md`）の「スクリプト一覧」テーブルに、今回追加したスクリプトの行（フォルダパスと概要）を追記する。
