# RR_to_SS_UUcounter

## 概要

楽楽販売の問い合わせDBからデータを取得し、**Gemini AI（Google）** を使って各問い合わせの商材カテゴリ・問い合わせ種別を自動分類します。分類結果をGoogleスプレッドシートの日別シート（UU件数）と全体シート（月別詳細集計）に書き込むスクリプトです。

---

## 処理の流れ

```
楽楽販売（CSVエクスポート）
   ↓ 前日の問い合わせデータを取得（最大10,000件）
データ整形
   ↓ 都道府県・対応履歴をパース
Gemini API（AI分類）
   ↓ 20件ずつバッチ処理で商材カテゴリ・サブカテゴリ・都道府県をJSON出力させる
集計
   ↓ 商材×都道府県×日別でUU数をカウント
Googleスプレッドシート更新
   ↓ 日別シート（UUのみ）と全体シート（詳細）を更新
```

---

## 詳細な処理内容

### 1. 楽楽販売からのデータ取得
- DB: 問い合わせ管理（dbSchemaId: 101181）
- 取得カラム: 記録ID、登録日、都道府県名、住所、新規商品タイプ、対応状況ステータス、対応履歴

### 2. Gemini AIによる自動分類

**商材カテゴリ（category）**
- 給湯器 / エコキュート / コンロ / その他商品 / 不明

**問い合わせ種別（sub_category）**
- `UU`：新規問い合わせ（ステータスが「1」で始まる、または特例パターン）
- `修理`：修理対応依頼
- `止`：緊急（お湯が出ないなど）
- `その他電話`：重複・クレーム・再工事など
- `その他`：除外対象（不可・エリア外・テスト等）

特例ルール（最優先）:「【交換工事に含まれるもの】」「新設商品」が履歴にある場合は必ずUU判定。

### 3. スプレッドシート更新

**日別シート（月名シート例：「1月」）**
- UU件数を都道府県×商材（給湯器/エコキュート）のマトリクスで書き込み

**全体シートの更新**
- UU/修理/止/その他電話の月別件数を加算更新

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GEMINI_API_KEY` | Google Gemini APIキー |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GCPサービスアカウントのJSON文字列 |

---

## 設定値（スクリプト内）

| 変数 | 内容 |
|---|---|
| `SPREADSHEET_KEY` | 書き込み先スプレッドシートID |
| `GEMINI_MODEL_NAME` | 使用するGeminiモデル名 |
| `DB_INQUIRY` | 楽楽販売のDB・検索・一覧ID設定 |

---

## 実行方法

```bash
python RR_to_SS_UUcounter-master.py
```

> `RR_to_SS_UUconter-sonotanodenwa.py` は「その他の電話」専用の別集計バージョンです。
