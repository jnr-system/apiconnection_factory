# RR_to_SS_uriage-arari

## 概要

楽楽販売から売上・粗利・原価等のデータを複数DB（正直屋・楽天L-stage・日本設備JERCY）からCSV形式でエクスポートし、施工日ベースおよび成約日ベースでそれぞれ日別集計したのち、Googleスプレッドシートの指定セルへ自動書き込みするスクリプトです。Google Sheets APIのレート制限（429エラー）対策としてExponential Backoffによる自動リトライを実装しています。

---

## ファイル構成

| ファイル | 説明 |
|---|---|
| `RR_to_SS_uriage-arari_combine.py` | メインスクリプト（3DB統合版） |
| `execution_csv_log.txt` | 実行ログ（自動生成・追記） |
| `.env_template` | ローカル開発用環境変数テンプレート |

---

## 処理の流れ

```
楽楽販売（CSVエクスポートAPI）
   ↓ 正直屋・楽天(L-stage)・日本設備(JERCY) の各DBからデータ取得（最大5,000件）
施工日ベース集計
   ↓ 施工日（またはメーカー出荷日）を基準に売上・原価・粗利・件数を日別・カテゴリ別に集計
成約日ベース集計（正直屋のみ詳細）
   ↓ 成約日を基準に件数・金額・キャンセル数・完了数・次月繰越・再工事数等を集計
Googleスプレッドシート更新
   ↓ 対象月の YYYYMM 形式シートを開き、指定セルへ日付列単位で書き込み
   ↓ 過去日は実績セル、今日以降の未来日は予定セルに分けて書き込み
   ↓ APIレート制限（429）時はExponential Backoffで自動リトライ
```

---

## 詳細な処理内容

### 1. DB設定と対象データ

3つのDBそれぞれに設定が定義されており、`DB_CONFIGS` リストで管理しています。

| DB名 | dbSchemaId | 集計軸 |
|---|---|---|
| 正直屋 | 101185 | 施工日 + 成約日（詳細集計あり） |
| 楽天(L-stage) | 101357 | メーカー出荷日 |
| 日本設備(JERCY) | 101378 | メーカー出荷日 |

更新対象期間: **前日〜31日後** の日付範囲

### 2. 施工日ベース集計

- 日付が対象期間内のレコードのみ集計
- 同一手配番号の重複は除去（`processed_daily_ids` で管理）
- 商品タイプ列のキーワードでカテゴリ分類（給湯器 / エコキュート / コンロ）
- カテゴリごとに `billing`（売上）・`cost`（原価）・`const`（粗利）・`count`（件数）を集計

### 3. 成約日ベース集計

正直屋の成約データは以下の通り分類されます：

| 分類 | 条件 |
|---|---|
| キャンセル | ステータスに「キャンセル」を含む（最優先） |
| 再工事（当月） | 依頼内容=「再工事」かつ施工日が成約月と同月 |
| 再工事（翌月） | 依頼内容=「再工事」かつ施工日が成約月より後 |
| 当月完了 | 施工日が成約月と同月 |
| 翌月繰越 | 施工日が成約月より後 |
| 未定 | 施工日が未記入 |

楽天・日本設備は依頼内容が「施工」「」「施工不可」のものだけを集計対象とします。

### 4. スプレッドシート書き込み

**シート名**: 対象月の `YYYYMM` 形式（例: `202501`）

**過去日・未来日の分離**:
- 今日より前の日付 → 実績セル（`cells` で指定）
- 今日以降の日付 → 予定セル（`future_cells` で指定）

**数値フォーマット**:
- 金額系: `#,##0,`（千の単位で表示）
- 件数系: `#,##0`（通常のカンマ区切り）

**書き込みセル（正直屋の主要項目）**:

| 項目 | セル番地 |
|---|---|
| 成約件数（合計） | AK12 列起点 |
| 成約金額（合計） | AK13 列起点 |
| キャンセル数 | AK14 列起点 |
| 当月完了数 | AK16 列起点 |
| 翌月繰越数 | AK17 列起点 |
| 再工事（当月） | AK18 列起点 |
| 再工事（翌月） | AK19 列起点 |
| 未定数 | AK20 列起点 |
| 給湯器・売上 | AK47 列起点 |
| エコキュート・売上 | AK75 列起点 |
| コンロ・売上 | AK103 列起点 |

楽天(L-stage) は `AK164〜166`、日本設備(JERCY) は `AK168〜170` に書き込みます。

### 5. 429エラー対策（Exponential Backoff）

Google Sheets APIのレート制限（Quota exceeded）を検知した場合:
- 最大5回リトライ
- 待機時間: 3秒 → 4秒 → 6秒 → 10秒 → 18秒（`2^i + 2` 秒）
- 429以外のエラーはリトライせずにスキップ

通常の書き込み後も2秒のスリープを入れてレート制限を回避しています。

## 設定項目（スクリプト内の主な定数）

| 変数名 | 内容 |
|---|---|
| `RAKURAKU_DOMAIN` | 楽楽販売ドメイン名 |
| `SPREADSHEET_KEY` | 書き込み先スプレッドシートキー |
| `DB_CONFIGS` | 取得対象DBのスキーマID・ビューID・セル番地などの設定リスト |
| `THOUSAND_FORMAT` | 金額用書式（`#,##0,`） |
| `NORMAL_FORMAT` | 件数用書式（`#,##0`） |
| `LOG_FILE_NAME` | ログファイル名（`execution_csv_log.txt`） |

---

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GCPサービスアカウントのJSON文字列 |

> ローカル開発時は `.env_template` を `.env` にリネームして環境情報を記入してください。

## 実行方法

```bash
python RR_to_SS_uriage-arari_combine.py
```

実行結果はコンソールおよび `execution_csv_log.txt` に出力されます。
