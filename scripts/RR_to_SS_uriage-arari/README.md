# RR_to_SS_uriage-arari

## 概要

楽楽販売から売上・粗利・原価等のデータを複数DB（正直屋・楽天・日本設備等）からCSV形式でエクスポートし、日付ベースおよび成約日ベースでそれぞれ集計したのち、Googleスプレッドシートの指定セルに自動書き込みを行うスクリプトです。APIのレート制限（429エラー）対策のための自動リトライ処理も組み込まれています。

---

## 処理の流れ

```text
楽楽販売（CSVエクスポート）
   ↓ 正直屋、楽天(L-stage)、日本設備(JERCY)などの各設定DB・ビューからデータを取得（最大5000件）
データ集計
   ↓ 施工日・成約日を基準として、売上・原価・粗利・件数などを全体・商品カテゴリごとに日別集計
Googleスプレッドシート更新
   ↓ 対象月のシート（例: 202404）を開き、対応する日付列に集計データを上書き更新（カンマ区切り等のフォーマット自動適用）
完了・ログ出力
```

---

## 詳細な処理内容

### データ取得と集計
- 楽楽販売 DB から「実行日の前日」〜「未来31日間分」を対象にCSV形式で随時エクスポート
- 日付（メーカー出荷日や施工日）などに紐付けて、「合計」「給湯器」「エコキュート」「コンロ」などのカテゴリごとに数値を日毎に積算計算
- 成約日を基準とした別のビューからもデータを取得し、重複や不要ステータスを除外した上での成約数や金額についても集計

### スプレッドシート書き込み
- 各日付が所属する月（YYYYMM形式）のシート名を検索・オープン
- 各DB・各集計カテゴリごとに指定された開始セル（例: AK17〜AK19 など）へ書き込み
- 過去の日付は実績セル、今日以降の未来日は別枠の予定セルに分けて記入する独自のロジック
- Google Sheets API レート制限（Quota exceeded: 429）を検知すると、待機時間を徐々に増やしながらのリトライ（Exponential Backoff）を実施

## 設定項目（スクリプト内の主な定数）

| 変数名 | 内容 |
|---|---|
| `RAKURAKU_DOMAIN` | 楽楽販売 ドメイン名 |
| `SPREADSHEET_KEY` | 書き込み先スプレッドシートキー |
| `DB_CONFIGS` | 取得対象DBのスキーマID、一覧、ビューID、キーワード定義や書込みセル番地を指定する辞書リスト |
| `LOG_FILE_NAME` | 処理ログの出力先（`execution_csv_log.txt`） |

---

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売の API トークン |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | Google Cloud サービスアカウントの JSON 文字列 |

> **Note:** ローカル開発や手動実行を行う場合は、同フォルダ内の `.env_template` を `.env` にリネーム・コピーし、自身の環境情報を設定して実行してください。

---

## 実行方法

```bash
python rakuraku_uriage_spreadsheet_api.py
```

実行結果やエラーのログは、コンソールに出力されるとともに同フォルダの `execution_csv_log.txt` にも保存されます。
