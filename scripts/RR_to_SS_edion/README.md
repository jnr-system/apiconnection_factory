# RR_to_SS_edion

## 概要

楽楽販売からエディオン案件の問い合わせデータを取得し、Googleスプレッドシートの「元データ管理シート（マスタDB）」に新規レコードのみ追記するスクリプトです。

スプレッドシートへの反映後、楽楽販売の「SS連携済み（エディオン）」フラグを `1` に更新することで、次回実行時に重複取得されないようにしています。

---

## 処理の流れ

```
楽楽販売（CSVエクスポートAPI）
   ↓ 「SS連携済み（エディオン）」が空のレコードのみ取得（最大5,000件）
   ↓ 指定列を抽出・整形（住所結合など）
Googleスプレッドシート
   ↓ 既存データの末尾の次の行のC列から追記
楽楽販売（レコード更新API）
   ↓ 反映済みレコードの「SS連携済み（エディオン）」を 1 に更新
完了ログ出力
```

---

## 詳細な処理内容

### 1. 楽楽販売からのデータ取得

- DB: 問い合わせ管理（`dbSchemaId: 101181`, `searchId: 107673`, `listId: 101473`）
- `searchId: 107673` は「SS連携済み（エディオン）が空」の絞り込み設定
- 最大5,000件を取得
- レスポンスは CP932 でデコード（失敗時は UTF-8 で再試行）

### 2. 列マッピングと整形

スプレッドシートのC列以降に以下の順序でデータを配置します：

| スプレッドシート列 | CSV列名 | 備考 |
|---|---|---|
| C列: レコードID | 記録ID | |
| D列: エディオン連携用手配番号 | エディオン連携用手配番号 | |
| E列: 問い合わせ日 | 新規問い合わせ日 | |
| F列: 顧客名 | 名前 | |
| G列: 顧客名カナ | フリカナ | |
| H列: 顧客電話番号 | 電話番号_1 | |
| I列: 顧客メールアドレス | メールアドレス_1 | |
| J列: 顧客住所 | 郵便番号+都道府県+市区+町村+マンション | 5列を結合 |
| K列: （空白） | ― | |
| L列: ガス種 | ガスの種類 | |
| M列: 先方店舗名 | エディオン店舗名 | |
| N列: 先方担当者名 | エディオン担当者名 | |
| O列: 工事第１希望日 | 第１希望日（エディオン） | |
| P列: 工事第２希望日 | 第２希望日（エディオン） | |
| Q列: 工事第３希望日 | 第３希望日（エディオン） | |
| R列: スグフォームURL | スグフォームURL（エディオン） | |

**住所結合ロジック**: 以下の5列を順に結合（空のフィールドはスキップ）
1. `エディオン案件用：郵便番号`
2. `都道府県名`
3. `市区名`
4. `町村名・番地`
5. `マンション・ビル名`

**データクリーニング**: 全セルに対して `strip()` + `lstrip("'")` を実施

### 3. スプレッドシートへの追記

- A列・B列（手動管理列）および1行目（ヘッダー行）は変更しない
- `get_all_values()` で現在の最終行を取得し、次の行のC列から追記

### 4. 楽楽販売のフラグ更新

- 項目ID `115979`（SS連携済み（エディオン））を `1` に更新
- 連続アクセス防止のため1件ごとに **1秒のスリープ**

## ファイル構成

| ファイル | 説明 |
|---|---|
| `RR_to_SS_edion.py` | メインスクリプト |
| `rr-ss-edion.service` | systemd サービスユニットファイル |
| `rr-ss-edion.timer` | systemd タイマーユニットファイル（15分毎） |
| `execution_log.txt` | 実行ログ（自動生成・追記） |
| `.env` | ローカル開発用の環境変数ファイル（Git管理外） |

---

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GCPサービスアカウントのJSON文字列 |

> ローカル実行時は同フォルダの `.env` ファイルに記述してください。
> 本番環境では `/etc/apiconnection.env` に設定してください。

---

## 実行方法

```bash
python RR_to_SS_edion.py
```

ログは同フォルダの `execution_log.txt` に追記されます。

---

## systemd による定期実行（Linux サーバー）

```bash
# ファイルをコピー
sudo cp rr-ss-edion.service /etc/systemd/system/
sudo cp rr-ss-edion.timer /etc/systemd/system/

# デーモン再読み込み・有効化・起動
sudo systemctl daemon-reload
sudo systemctl enable rr-ss-edion.timer
sudo systemctl start rr-ss-edion.timer

# 状態確認
sudo systemctl status rr-ss-edion.timer
```
