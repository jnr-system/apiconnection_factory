# RR_to_SS_seiyakuprocess

## 概要

楽楽販売の問い合わせ管理DBから案件データを取得し、**緊急度別（急ぎ・故障・不具合検討）・商品タイプ別（給湯器・エコキュート）** に KPI（UU数・成約数・終了数）を集計してGoogleスプレッドシートへ書き込むスクリプトです。

スナップショット機能により、過去データを保持して正確な履歴ベースの集計が可能です。

---

## ファイル構成

| ファイル | 説明 |
|---|---|
| `RR_to_SS_seiyakuprocess-joukyoustatus.py` | メインスクリプト（対応状況ステータスベース） |
| `RR_to_SS_seiyakuprocess-kinkyustatus.py` | 緊急度ステータスベースの別バージョン |
| `snapshots/` | 日次スナップショットJSONの保存フォルダ |

---

## 処理の流れ

```
楽楽販売（CSVエクスポート）
   ↓ 問い合わせ管理DBの全件を取得
スナップショット保存（YYYY-MM-DD.json）
   ↓ 当日分のスナップショットとして保存（3ヶ月より古いファイルは自動削除）
過去スナップショット読み込み
   ↓ 集計対象日のスナップショットがあれば正確な過去データで集計、なければ最新データで補完
KPI集計（緊急度×商品タイプ×単日/累計）
   ↓ UU数・成約数・終了数を緊急度別に集計
Googleスプレッドシート更新
   ↓ 月別シートへ単日値・累積値（過去90日）を書き込み
```

---

## 詳細な処理内容

### KPI 定義

| KPI | 条件 |
|---|---|
| UU数 | ステータス = `1 ■一次受付（ヒアリング）` |
| 成約数 | ステータス = `1-50 ■成約` |
| 終了数 | ステータスに「失注」を含む |

### 緊急度分類

`急ぎ` / `故障` / `不具合・検討` の3種別で分類し、それぞれ単日・累計（過去90日）を算出

### スプレッドシート書き込み位置

- 給湯器: `COL_START_KYUTOKI`（AP列）起点
- エコキュート: `COL_START_ECO`（BZ列）起点
- 全体集計: `COL_START_ALL`（F列）起点
- 日付ごとに列をシフト（1日=基準列、2日=+1列…）

## 実行方法

```bash
python RR_to_SS_seiyakuprocess-joukyoustatus.py
```

デフォルトでは前日を最終日として **過去30日分** を集計・更新します。  
特定期間を指定する場合はスクリプト内の `TARGET_DATE_START` / `TARGET_DATE_END` を設定してください。
