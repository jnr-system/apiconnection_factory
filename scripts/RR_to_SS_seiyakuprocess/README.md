# RR_to_SS_seiyakuprocess

## 概要

楽楽販売の問い合わせ管理DBと成約管理DBからデータを取得し、KPIを集計してGoogleスプレッドシートへ書き込むスクリプト群です。

| スクリプト | 集計軸 |
|---|---|
| `RR_to_SS_seiyakuprocess-joukyoustatus.py` | 対応状況ステータス別・商品タイプ別 |
| `RR_to_SS_seiyakuprocess-kinkyustatus.py` | 緊急度別・商品タイプ別・ステータス小項目別 |

---

## joukyoustatus.py の処理内容

### データソース
- 問い合わせ管理DB（`dbSchemaId=101181`）: searchId `107226`, `107228`
- 成約管理DB（`dbSchemaId=101185`）: searchId `107230`, `107239`

### 処理フロー

```
楽楽販売（CSVエクスポートAPI）
   ↓ 問い合わせ管理DB・成約管理DBを取得（重複除去）
KPI集計
   ↓ ステータス別 × 商品タイプ別 で単日（当日0時〜実行時刻）・累計（集計日の15日前〜集計日）を集計
Googleスプレッドシート更新
   ↓ 月別シートへ単日値・累積値を書き込み
```

### ステータス正規化 (`normalize_status()`)

| ルール | 処理 |
|---|---|
| `STATUS_ROW_MAP` に定義済みのステータス | そのまま返す |
| `1-62` で始まるステータス | `1-62 ■エリア外、施工・対応不可` に統一 |
| エディオン紹介 / FCへ紹介 / その他（問い合わせ以外の物） | `その他全て` に変換 |
| 上記以外 | `_対象外_`（集計から除外） |

### 成約管理DBの加工 (`process_contract_raw_data()`)

- 商品タイプ判定:
  1. `商品タイプ` 列に値があればそれを使用
  2. なければ金額で推定: 10〜30万円 → 給湯器、30万円以上 → エコキュート
- ステータス判定:
  - `成約日 == 問い合わせ日` → 当日成約
  - それ以外 → 当日外成約

### スプレッドシート書き込み位置

| 集計種別 | 開始列 |
|---|---|
| 全体 | F列（6列目） |
| 給湯器 | AQ列（43列目） |
| エコキュート | CB列（80列目） |

## kinkyustatus.py の処理内容

### データソース
- 問い合わせ管理DB（`dbSchemaId=101181`）のみ: searchId `107226`

### 処理フロー

```
楽楽販売（CSVエクスポートAPI）
   ↓ 問い合わせ管理DBを取得（重複除去）
KPI集計
   ↓ 緊急度別 × 商品タイプ別 × ステータス小項目別 で単日（当日のみ）・累計（集計日の15日前〜集計日）を集計
Googleスプレッドシート更新
   ↓ 月別シートへ単日値・累積値を書き込み
```

### 集計軸

3軸でデータを絞り込み：

| 軸 | パターン |
|---|---|
| 商品タイプ | 全体 / 給湯器 / エコキュート |
| 緊急度 | 急ぎ / 故障 / 不具合・検討 |
| 期間 | 単日（当日のみ） / 累計（集計日の15日前〜集計日） |

### ステータス小項目 (STATUS_GROUPS)

各緊急度の下に以下の小項目を集計：

| 小項目 | 対象ステータス |
|---|---|
| 一次受付 | `1 ■一次受付（ヒアリング）` |
| 概算見積 | `1-10 ■概算見積（写真フォーム案内なし）`, `1-11 ■概算見積（写真待ち）`, `1-20 ■確認中（施工可否、現調含む）`, `1-40 ■現地調査（成約前）` |
| 最終見積 | `1-30 ■最終見積り済` |
| 成約 | `当日成約`, `当日外成約` |
| 失注 | `1-61-A〜E ■失注` 5種 |
| キャンセル | `1-60 ■キャンセル（成約からキャンセル）` |
| 対応不可 | `1-62 ■エリア外、施工・対応不可` |

### スプレッドシート書き込み位置

| 集計種別 | 開始列 |
|---|---|
| 全体 | D列（4列目） |
| 給湯器 | AL列（38列目） |
| エコキュート | BT列（72列目） |

**累計の書き込み行（小項目順に連続）**

| 緊急度 | 行範囲 |
|---|---|
| 急ぎ | 7〜13行 |
| 故障 | 15〜21行 |
| 不具合・検討 | 23〜29行 |

**単日の書き込み行（小項目順に連続）**

| 緊急度 | 行範囲 |
|---|---|
| 急ぎ | 34〜40行 |
| 故障 | 42〜48行 |
| 不具合・検討 | 50〜56行 |

## 設定値（スクリプト内）

| 変数 | 内容 |
|---|---|
| `TARGET_DATE_START` / `TARGET_DATE_END` | 手動で集計期間を指定（空欄なら当日を自動設定） |
| `SPREADSHEET_KEY` | 書き込み先スプレッドシートID |
| `COL_START_ALL` | 全体集計の開始列 |
| `COL_START_KYUTOKI` | 給湯器集計の開始列 |
| `COL_START_ECO` | エコキュート集計の開始列 |

## 実行方法

```bash
# 対応状況ステータスベース
python RR_to_SS_seiyakuprocess-joukyoustatus.py

# 緊急度ステータスベース
python RR_to_SS_seiyakuprocess-kinkyustatus.py
```

デフォルトでは **当日** を集計対象とします（1時間ごとの定期実行を想定）。
特定期間を指定する場合はスクリプト内の `TARGET_DATE_START` / `TARGET_DATE_END` を設定してください。

累計は常に **集計日の15日前〜集計日** のローリング集計です。
`joukyoustatus.py` の単日は **当日0時〜実行時刻** までの時刻フィルタが適用されます。

## systemd による定期実行（Linux サーバー）

1時間毎に自動実行されます。

```bash
# ファイルをコピー
sudo cp rr-seiyakuprocess.service /etc/systemd/system/
sudo cp rr-seiyakuprocess.timer /etc/systemd/system/

# デーモン再読み込み・有効化・起動
sudo systemctl daemon-reload
sudo systemctl enable rr-seiyakuprocess.timer
sudo systemctl start rr-seiyakuprocess.timer

# 状態確認
sudo systemctl status rr-seiyakuprocess.timer
```
