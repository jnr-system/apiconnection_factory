# RR_to_SS_seiyakuprocess

## 概要

楽楽販売の問い合わせ管理DBと成約管理DBからデータを取得し、**対応状況ステータス別・商品タイプ別** に KPI（各ステータスの件数）を集計してGoogleスプレッドシートへ書き込むスクリプトです。

**スナップショット機能**により、過去データを日次JSONファイルとして保存することで、後から正確な履歴ベースの集計が可能です。

---

## ファイル構成

| ファイル | 説明 |
|---|---|
| `RR_to_SS_seiyakuprocess-joukyoustatus.py` | メインスクリプト（対応状況ステータスベース） |
| `RR_to_SS_seiyakuprocess-kinkyustatus.py` | 緊急度ステータスベースの別バージョン |
| `snapshots/` | 日次スナップショットJSONの保存フォルダ |
| `execution_log.txt` | 実行ログ（自動生成・追記） |

---

## 処理の流れ

```
楽楽販売（CSVエクスポートAPI）
   ↓ 問い合わせ管理DBの全件を取得（2つの searchId を順に取得・重複除去）
   ↓ 成約管理DBの全件を取得（2つの searchId を順に取得・重複除去）
スナップショット保存（YYYY-MM-DD.json）
   ↓ 当日分のスナップショットとして保存（記録ID→ステータス・商品タイプのマップ）
   ↓ 3ヶ月より古いスナップショットファイルを自動削除
過去スナップショット読み込み
   ↓ 集計対象日のスナップショットが存在すれば過去データで正確に集計
   ↓ 存在しなければ最新データで代替（フォールバック）
KPI集計
   ↓ ステータス別 × 商品タイプ別 で単日・累計（当月1日〜対象日）を集計
Googleスプレッドシート更新
   ↓ 月別シートへ単日値・累積値を書き込み
```

---

## 詳細な処理内容

### 1. データ取得 (`fetch_rakuraku_csv()`)

- 問い合わせ管理DB: `dbSchemaId=101181`、2つの searchId（`107226`, `107228`）を順に取得
- 成約管理DB: `dbSchemaId=101185`、2つの searchId（`107230`, `107239`）を順に取得
- 各DBとも最大10,000件取得、重複は記録IDで除去
- レスポンスは CP932 → UTF-8 フォールバックでデコード

### 2. ステータス正規化 (`normalize_status()`)

| ルール | 処理 |
|---|---|
| `STATUS_ROW_MAP` に定義済みのステータス | そのまま返す |
| `1-62` で始まるステータス | `1-62 ■エリア外、施工・対応不可` に統一 |
| エディオン紹介 / FCへ紹介 / その他（問い合わせ以外の物） | `その他全て` に変換 |
| 上記以外 | `_対象外_`（集計から除外） |

### 3. スナップショット管理

**保存 (`save_snapshot()`)**
- `snapshots/YYYY-MM-DD.json` として保存
- 内容: `{記録ID: {status, product_type, inq_date}}` の辞書

**読み込み (`load_snapshot_for_date()`)**
- 指定日のファイルがあればそれを読み込み
- ファイルがなければ `None` を返す（最新データで代替）

**自動削除 (`cleanup_old_snapshots()`)**
- 実行時に3ヶ月より前のスナップショットファイルを自動削除

### 4. 成約管理DBの加工 (`process_contract_raw_data()`)

- 商品タイプ判定:
  1. `商品タイプ` 列に値があればそれを使用
  2. なければ金額で推定: 10〜30万円 → 給湯器、30万円以上 → エコキュート
- ステータス判定:
  - `成約日 == 問い合わせ日` → 当日成約
  - それ以外 → 当日外成約

### 5. スプレッドシート書き込み (`update_spreadsheet_cells()`)

| 集計種別 | 書き込み位置の基準列 |
|---|---|
| 全体集計 | `COL_START_ALL = F列（6列目）` |
| 給湯器 | `COL_START_KYUTOKI = AQ列（43列目）` |
| エコキュート | `COL_START_ECO = CB列（80列目）` |

- 各基準列から `(対象日 - 1)` 列ずらした列に書き込み（例: 5日なら+4列）
- 単日はその日のスナップショットから集計
- 累計は当月1日〜対象日のスナップショットから集計

**単日の書き込み行 (STATUS_ROW_MAP)**

| ステータス | 行番号 |
|---|---|
| 1 ■一次受付（ヒアリング） | 46 |
| 1-10 ■概算見積（写真フォーム案内なし） | 48 |
| 1-20 ■確認中 | 50 |
| 1-30 ■最終見積り済 | 51 |
| 当日成約 | 54 |
| 当日外成約 | 55 |
| 失注系（A〜E）・キャンセル | 57〜62 |
| 1-62 ■エリア外 | 64 |
| その他全て | 66 |

**累計の書き込み行 (CUMULATIVE_ROW_MAP)**

| ステータス | 行番号 |
|---|---|
| 1 ■一次受付（ヒアリング） | 8 |
| 概算見積系 | 10〜11 |
| 確認中〜現地調査 | 12〜14 |
| 当日成約・当日外成約 | 16〜17 |
| 失注系・キャンセル | 19〜24 |
| エリア外 | 26 |
| その他全て | 28 |

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GCPサービスアカウントのJSON文字列 |

---

## 設定値（スクリプト内）

| 変数 | 内容 |
|---|---|
| `TARGET_DATE_START` / `TARGET_DATE_END` | 手動で集計期間を指定（空欄なら前日〜3日前を自動設定） |
| `SPREADSHEET_KEY` | 書き込み先スプレッドシートID |
| `SNAPSHOT_DIR` | スナップショット保存先（`snapshots/` フォルダ） |
| `COL_START_ALL` | 全体集計の開始列（F列=6） |
| `COL_START_KYUTOKI` | 給湯器集計の開始列（AQ列=43） |
| `COL_START_ECO` | エコキュート集計の開始列（CB列=80） |

---

## 実行方法

```bash
# 対応状況ステータスベース
python RR_to_SS_seiyakuprocess-joukyoustatus.py

# 緊急度ステータスベース
python RR_to_SS_seiyakuprocess-kinkyustatus.py
```

デフォルトでは前日〜3日前の **過去3日間** を集計・更新します。
特定期間を指定する場合はスクリプト内の `TARGET_DATE_START` / `TARGET_DATE_END` を設定してください。
