# RR_to_SS_zenkokuseiyaku

## 概要

楽楽販売の成約管理DBから成約データを取得し、**商品タイプ（給湯器・エコキュート・コンロ・その他）および都道府県別** に集計してGoogleスプレッドシートの月別シートと全体シートへ書き込むスクリプトです。

---

## 処理の流れ

```
楽楽販売（CSVエクスポートAPI）
   ↓ 成約管理DBから2つの検索条件で取得（先月分 + 今月分）・重複除去
商品タイプ判定（行単位）
   ↓ コンロキーワード → 商品タイプ列 → 型番パターン → 金額推定 の順で判定
都道府県判定
   ↓ 施工先都道府県列 → 住所文字列から抽出 → 「不明」
手配番号ごとに「主役商品」を決定
   ↓ エコキュート > 給湯器 > コンロ > その他 の優先順で1契約1商品に集約
Googleスプレッドシート更新
   ↓ 全体シート（月別合計）を上書き更新
   ↓ 月別シート（都道府県×日付のマトリクス）を更新
```

---

## 詳細な処理内容

### 1. データ取得 (`fetch_rakuraku_csv()`)

- DB: 成約管理（`dbSchemaId: 101185`, `listId: 101454`）
- 2つの searchId（`107200`=先月用、`107191`=今月用）を順に取得し、`concat` + `drop_duplicates` で重複除去
- 最大10,000件/リクエスト、CP932 → UTF-8 フォールバックでデコード

### 2. 商品タイプ判定ロジック (`determine_row_product_type()`)

以下の優先順位で判定します（上位条件が一致した時点で決定）:

| 優先度 | 条件 | 判定結果 |
|---|---|---|
| 1（最優先） | コンロ系キーワード: `コンロ`, `N3W`, `RS31`, `PD-`, `RHS`, `N3S`, `ビルトイン` が商品タイプ・商品名に含まれる | コンロ |
| 2 | `商品タイプ` 列に「給湯器」を含む | 給湯器 |
| 2 | `商品タイプ` 列に「エコキュート」または「電気温水器」を含む | エコキュート |
| 3 | 商品名（大文字化）にエコキュート型番: `SRT-`, `EQ`, `HE-`, `BHP-`, `HWH-`, `CHP-`, `電気温水器` を含む | エコキュート |
| 3 | 商品名に給湯器型番: `GT-`, `RUF-`, `GQ-`, `GTH-`, `RVD-`, `RUX-`, `FH-`, `PH-` を含む | 給湯器 |
| 4 | 金額 ≥ 30万円 | エコキュート |
| 4 | 金額 ≥ 10万円 | 給湯器 |
| 5（デフォルト） | 上記のいずれにも該当しない | その他 |

### 3. 都道府県判定 (`determine_prefecture()`)

1. `（日程調整）施工先都道府県` 列に都道府県名があればそれを使用
2. なければ `（日程調整）住所` フィールドの文字列内に都道府県名が含まれるか確認
3. 判定不能な場合は「不明」

### 4. 手配番号ごとの集約 (`process_sales_data()`)

同一手配番号に複数の商品行が含まれる場合、以下の優先順位で1商品に集約します:

**エコキュート > 給湯器 > コンロ > その他**

```python
def decide_contract_type(group):
    types = group["行の推測"].tolist()
    if "エコキュート" in types: return "エコキュート"
    if "給湯器"    in types: return "給湯器"
    if "コンロ"    in types: return "コンロ"
    return "その他"
```

### 5. スプレッドシート更新 (`update_spreadsheet_cells()`)

**全体シート（`全体`）**
- 月ごと・商品タイプごとの件数を上書き更新（二重計上防止のため加算ではなく上書き）
- 行設定: 給湯器=4行目、エコキュート=5行目、コンロ=6行目、その他=7行目
- 列設定: 1月=C列(3)、2月=D列(4)... （`col_idx = month + 2`）

**月別シート（例: `1月`）**
- 給湯器: C列（`COL_START_GAS=3`）起点、エコキュート: AI列（`COL_START_ECO=35`）起点
- 6行目〜: 都道府県順（北海道〜沖縄・不明）
- 日付 N 日は起点列から (N-1) ずれた列に書き込み
- 件数0は `"-"`、1以上は整数値で書き込み

## 設定値（スクリプト内）

| 変数 | 内容 |
|---|---|
| `MANUAL_TARGET_DATE` | 手動で基準日を指定（例: `"2025/01/15"`、空欄なら前日自動） |
| `DAYS_TO_SYNC` | 集計日数（デフォルト: `1` 日） |
| `SPREADSHEET_KEY` | 書き込み先スプレッドシートID |
| `DB_SEIYAKU` | 楽楽販売 成約管理DBの設定（スキーマID・searchId・列名） |
| `COL_START_GAS` | 給湯器の1日目の列番号（3=C列） |
| `COL_START_ECO` | エコキュートの1日目の列番号（35=AI列） |
| `START_ROW` | 都道府県データ開始行（6行目） |
| `OVERALL_ROWS` | 全体シートの商品タイプ別行番号 |

---

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GCPサービスアカウントのJSON文字列 |
