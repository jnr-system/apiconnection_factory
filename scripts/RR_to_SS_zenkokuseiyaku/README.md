# RR_to_SS_zenkokuseiyaku

## 概要

楽楽販売の成約管理DBから成約データを取得し、**商品タイプ（給湯器・エコキュート・コンロ・その他）および都道府県別** に集計してGoogleスプレッドシートの月別シートと全体シートへ書き込むスクリプトです。

---

## 処理の流れ

```
楽楽販売（CSVエクスポート）
   ↓ 成約管理DBから2つの検索条件で取得（先月分＋今月分）・重複除去
商品タイプ判定
   ↓ 商品タイプ列・商品名・型番・金額から給湯器/エコキュート/コンロ/その他を決定
都道府県判定
   ↓ 施工先都道府県列 → 住所から都道府県名を抽出 → 不明
手配番号ごとに「主役商品」を決定
   ↓ エコキュート > 給湯器 > コンロ > その他の優先順で1件に集約
Googleスプレッドシート更新
   ↓ 月別シート（都道府県×日付）と全体シート（月別合計）を書き込み
```

---

## 詳細な処理内容

### 商品タイプ判定ロジック（優先順位順）

1. コンロ関連キーワード（型番：N3W, RS31, PD-, RHS など）
2. `商品タイプ` 列に「給湯器」「エコキュート」を含む
3. `商品名` の型番パターン（SRT-, GT-, RUF-, HE- など）で判定
4. 金額で推定（30万円以上 → エコキュート、10万円以上 → 給湯器）

### スプレッドシート書き込み位置

**月別シート（例: `1月`）**
- 給湯器：C列（3列目）起点、都道府県行 × 日付列
- エコキュート：AI列（35列目）起点

**全体シート**
- 月列（1月=C、2月=D...）へ商品タイプ別の月間成約合計を上書き

## 設定値（スクリプト内）

| 変数 | 内容 |
|---|---|
| `MANUAL_TARGET_DATE` | 手動で基準日を指定（空欄なら前日自動） |
| `DAYS_TO_SYNC` | 集計日数（デフォルト: 1日） |
| `SPREADSHEET_KEY` | 書き込み先スプレッドシートID |
| `DB_SEIYAKU` | 楽楽販売 成約管理DBの設定 |

---

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GCPサービスアカウントのJSON文字列 |

---

## 実行方法

```bash
python RR_to_SS_zenkokuseiyaku.py
```
