# RR_to_RMS_customerdata

## 概要

楽楽販売に登録された「待機中の注文」に対して、楽天RMS（受注管理システム）から対応する注文情報を取得し、顧客情報・配送先情報を楽楽販売のレコードへ自動転記するスクリプトです。

---

## 処理の流れ

```
楽楽販売（CSVエクスポート）
   ↓ 「待機中」ステータスのレコードを取得（楽天注文番号 → 楽楽キーID のマップを作成）
楽天RMS API（注文検索・詳細取得）
   ↓ 直近3日分の注文番号を検索し、100件ずつ詳細情報を取得
楽楽販売（レコード更新API）
   ↓ 一致した注文の顧客情報・配送先・金額・配送希望日を楽楽販売へ書き込み
完了ログ出力
```

---

## 詳細な処理内容

### 1. 楽楽販売から待機データを取得
- CSVエクスポートAPIを使用し、対象DBから最大1,000件を取得
- 1列目：楽楽販売のキーID、2列目：楽天注文番号 として記録する `{注文番号: キーID}` のMapを作成

### 2. 楽天RMS APIから注文情報を取得
- 過去3日間の注文を検索し、注文番号の一覧を取得（最大4ページ）
- 注文番号を100件ずつに分割して詳細情報を取得

### 3. 楽楽販売レコードの更新
楽楽販売と楽天注文番号が一致した場合、以下のフィールドを更新：

| フィールドID | 内容 |
|---|---|
| 113811 | 注文者の電話番号 |
| 113772 | 注文合計金額 |
| 113095 | 配送先郵便番号 |
| 113073 | 配送希望日（備考欄から正規表現で抽出） |
| 113096 | 配送先都道府県 |
| 113097 | 配送先市区町村 |
| 113098 | 配送先番地・建物名 |
| 113089 | 配送先電話番号 |

連続アクセス防止のため、1件更新ごとに **1秒のスリープ** を入れています。

---

## 環境変数（必須）

| 変数名 | 説明 |
|---|---|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `RMS_SERVICE_SECRET` | 楽天RMS サービスシークレット |
| `RMS_LICENSE_KEY` | 楽天RMS ライセンスキー |

---

## 実行方法

```bash
python RR_to_RMS_customerdata.py
```

ログは同フォルダの `execution_log.txt` に追記されます。
